API

The API exposes platform-level tenant management endpoints. Tenant-scoped endpoints should include the X-Tenant-Slug header unless the path is explicitly skipped by tenant resolution.

Tenant Catalog

Create Tenant

POST /api/v1/tenants
Content-Type: application/json

{
  "name": "Acme",
  "slug": "acme",
  "culture": "en-US"
}

Creates a catalog tenant record with Pending status and returns tenant metadata. This route does not provision a tenant database.

Get Tenant By Slug

GET /api/v1/tenants/acme

Returns a tenant by slug, or 404 Not Found when no tenant exists.

List Tenants

GET /api/v1/tenants

Returns all catalog tenants visible through the tenant repository.

Tenant Operations

Onboard Tenant

POST /api/admin/tenants/onboard
Content-Type: application/json

{
  "name": "Acme",
  "slug": "acme"
}

Creates the catalog tenant, creates the tenant database, applies tenant migrations, and marks the tenant Active or Failed.

Upgrade Tenant

POST /api/admin/tenants/{id}/upgrade

Runs tenant database migrations for the tenant identified by id. Returns a migration result with the database name, success flag, applied migration count, duration, and optional error.

Soft Delete Tenant

DELETE /api/admin/tenants/{id}
Content-Type: application/json

{
  "reason": "Customer cancellation"
}

Marks the tenant as deleted in the catalog. The tenant database is not dropped during soft deletion.

Hard Delete Tenant

POST /api/admin/tenants/{id}/hard-delete

Drops the tenant database and removes the catalog record. The tenant must already be soft-deleted.
